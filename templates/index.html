What is Keystroke Dynamics?

Keystroke dynamics is a behavioural biometric — it identifies users based on their unique typing rhythm. Unlike passwords, it is continuous and invisible to the user.

Each person has a characteristic pattern of:

  • Hold time — how long each key is pressed
  • Flight time — gap between key-up and next key-down
  • Digraph latency — timing of consecutive key pairs
  • Typing speed — words per minute rhythm
  • Error patterns — backspace frequency & placement
System Architecture
① Browser captures keydown/keyup timestamps
② 16 timing features extracted per session
③ Classical ML: SVM · Random Forest · k-NN
④ Deep Learning: MLP / LSTM architecture
⑤ Anomaly Detection: Autoencoder + OCSVM
⑥ Majority vote → Authenticated / Rejected
Dataset

Synthetically generated dataset with 10 simulated users, each with unique timing distributions (mean hold time 78–155ms, mean flight 65–135ms).

1,000 labelled sessions — 50% static password, 50% free-text. Models trained with 5-fold stratified cross-validation.

Metrics Explained
  • Accuracy — correct identity predictions / total
  • ROC-AUC — area under receiver operating curve
  • FAR — false acceptance rate (impostors let in)
  • FRR — false rejection rate (genuine users blocked)
  • EER — equal error rate (FAR = FRR point, lower = better)
