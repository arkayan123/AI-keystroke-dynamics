# KeyAuth — Keystroke Dynamics for Continuous Authentication
### MSc Cybersecurity Project

---

## Project Overview

This project implements a **full keystroke dynamics authentication system** using three ML paradigms:

| Approach | Models | Accuracy |
|---|---|---|
| Classical ML | SVM (RBF), Random Forest, k-NN | **81.6%** (RF) |
| Deep Learning | LSTM / MLP | **78.5%** val acc |
| Anomaly Detection | One-Class SVM, Isolation Forest, Autoencoder | EER 20%, AUC 0.91 |

---

## Quick Start

### Option A — Open the standalone demo (no install needed)
```
Open KeyAuth_Demo.html in any browser
```
- Live keystroke capture with real timing extraction
- In-browser classifier using Gaussian profile matching  
- Simulator tab to generate genuine/impostor sessions
- Model results and feature analysis tabs

### Option B — Run the full Python backend

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Train all models
python train.py

# 3. Launch the Flask web app
python app.py
# Then open http://localhost:5050
```

## Dataset

Synthetically generated dataset simulating 10 users with distinct typing profiles:

| Feature | Description |
|---|---|
| `hold_mean/std` | Mean & std of key press duration (ms) |
| `flight_mean/std` | Mean & std of inter-key gap (ms) |
| `digraph_mean/std` | Mean & std of consecutive key timing |
| `wpm` | Words per minute |
| `backspace_rate` | Error correction frequency |
| `total_time_ms` | Session duration |

Users range from 2.8 cps (slow) to 6.2 cps (fast) with unique timing distributions.

---

## Features Extracted (16 total)

- Hold time: mean, std, min, max
- Flight time: mean, std, min, max  
- Digraph latency: mean, std, min, max
- Typing speed (WPM)
- Backspace rate
- Total session time
- Keystroke count

## Evaluation Metrics

- **Accuracy** — correct identity predictions (5-fold CV)
- **ROC-AUC** — multi-class one-vs-rest  
- **FAR** — False Acceptance Rate (impostors let in)
- **FRR** — False Rejection Rate (genuine users blocked)
- **EER** — Equal Error Rate (FAR = FRR operating point)

3. **Drift handling** — add online learning / sliding window retraining

5. **Federated learning** — on-device models for privacy preservation
